## INFORMATION PRIORITY (Visual Hierarchy)

### TODAY Screen Hierarchy
```
1. Company Name (Largest, black, h1)
   └─ What you're contacting

2. Challenge (Medium, dark gray, body)
   └─ Why you're contacting

3. Approach (Small, gray, body)
   └─ How you're approaching

4. CTA Buttons (Large, action color)
   └─ What to do now

5. Secondary link (Small, low contrast)
   └─ If you need more info
```

### CONVERSATIONS Screen Hierarchy
```
1. Company Name (Medium, black, h2)
   └─ Context for timeline

2. Interaction Date (Small, gray, meta)
   └─ When it happened

3. Interaction Type (Small, dark gray, meta)
   └─ What kind of interaction

4. Subject/Note (Small, dark gray, body)
   └─ Key information about interaction

5. Status (Tiny, gray, meta)
   └─ Engagement status (opened, replied, etc.)
```

### OPPORTUNITIES Screen Hierarchy
```
1. Company Name (Medium, black, h2)
   └─ Which standing order

2. Frequency (Small, gray, meta)
   └─ How often

3. Dates (Small, gray, meta)
   └─ Last and next

4. Response Rate (Small, dark gray, body)
   └─ How it's working

5. Actions (Tiny, low contrast)
   └─ What you can do
```

## DESIGN CONSTRAINTS

✅ **Applied:**
- Monochrome (black, white, gray only)
- No badges
- No colored backgrounds
- No CRM language
- No scores displayed
- No tier labels
- Minimal borders (dividers only)
- Whitespace > information
- 3 typography sizes maximum
- One primary action per screen
- Progressive disclosure (expand for details)

❌ **Avoided:**
- Colored cards
- Dashboard metrics
- Complex navigation
- Competing CTAs
- Dense information grids
- Visual noise
- Multiple decision points per screen

---

## MOBILE PRINCIPLES

- All content stacks vertically
- Buttons full-width
- 375px max width (iPhone 12 mini)
- Touch targets ≥ 44px
- Tap-friendly spacing
- Thumb-zone optimized (buttons at bottom)
- Readable without pinch-zoom
- Landscape not optimized (portrait primary)

---

## RESPONSIVE BREAKPOINTS

```
Mobile (375px):     Stacked, full-width buttons
Tablet (768px):     2-column grid, buttons side-by-side
Desktop (1024px+):  Centered max-width (600px), whitespace

All screens maintain:
- Same information hierarchy
- Same visual design
- Same typography sizes
- Same action placement
```

---

## KEY DIFFERENCES FROM CURRENT STATE

**Current (/b2b/leads):**
- Grid of cards (2-column)
- Multiple information points per card
- Tier labels visible
- Filter UI at top
- Dense, information-first

**Future (TODAY):**
- Single card (full width)
- Minimal information (company, challenge, approach)
- No tier labels
- No filter UI
- Action-first, minimal
